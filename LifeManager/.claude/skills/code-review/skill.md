## Instructions

1. Lance `git diff --name-only HEAD~1` pour identifier les fichiers modifiés récemment.
2. Lis chaque fichier modifié.
3. Pour chaque fichier, analyse selon les 4 priorités ci-dessous.
4. Produis le rapport uniquement si tu as trouvé des problèmes. Si une priorité n'a aucun problème, ne l'affiche pas.
